# AIPulse

AIPulse is an open-source system to **measure, govern, and scale AI adoption** inside teams. It’s built to prevent “AI work‑intensification” by tracking real impact, quality, and risk—while enabling agentic workflows to operate inside the dashboard.

## Why
AI adoption often adds invisible work: review cycles, QA, coordination, and new process overhead. AIPulse makes that visible and manageable.

## Core Modules
- **Adoption & Usage**: active users, tasks assisted, frequency.
- **Impact**: time saved vs. review cost; KPI changes.
- **Quality & Risk**: retrabajo, compliance flags, sensitive data controls.
- **Case Management**: prioritized use cases with owners and success metrics.
- **Agentic Ops**: agents that collect signals, produce reports, and raise alerts.

## MVP Scope (Phase 1)
1) Dashboard with core KPIs
2) Use‑case registry (3–5 prioritized use cases)
3) Monthly report agent
4) Minimal policy & governance layer
